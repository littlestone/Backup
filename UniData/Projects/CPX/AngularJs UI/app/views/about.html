Project CPX – Capex Management System - Release notes

 

1. Introduction

CPX is the future system for CAPEX Management that will allow users to

•       Create Projects (TEMPs)

•       Create AFE related to projects with approval workflow (SLA)

•       Track AFE and POs associated to Projects; as well as all related Capital expenditure.

•       Input Budget for CAPEX

•       Manage Forecast for  CAPEX Projects

•       Create traceability reports

•       Allow for Assets as identified by Equipment types to be linked to GL categories and associated to AFE and projects

 

Nomenclature in CPX:

Project = TEMP

AFE = Authorization for Expenditure

 

2. Milestone 2 features presentation:

Currently, projects are stored in TM1.

In order to align both systems, projects were uploaded out of TM1 into CPX:

-          Current open TEMPs and AFE;

-          Projects closed in 2016;

-          Including phasing of the Budget and Forecast, as well as actuals in relevant months;

 

2.1. Feature #1: Project List screen

·   The project List is a screen that enables the user to display a list of projects using selection criteria

-          By Project Owner / Location / Department

-          By Opened/closed project

·  Possibility to use the drop down list to display the list of AFE(s) per Project

·  One click on the Project# or AFE# opens the related detailed file

 

2.2. Feature #2: Project (TEMP) display screen

·  The section “Header” contains Project data from TM1. eg. aliaxis category or aliaxis #

·  The section “Phasing” allows the viewing of the phasing of the forecast and the actuals. A graph will display the phasing of the Budget as well as the Forecast and Actuals

Exception: other sections of the Project detailed file are blank and will be filled by the user. eg. Aliaxis#

2.3. Feature #3: AFE display

·  The section “Header” contains Project data from TM1. eg. Project number or amount in $

·  The section ‘Phasing” allows the viewing of the phasing of the forecast and the actuals. A graph will display the phasing of the Forecasts and actuals

Exception: other sections of the AFE detailed file are blank and will be filled either by the user or by an upload from Infoflo. Eg. CER#, WO#, Asset#, etc.

2.4. Feature #4: User authentication

Authorized users have access to CPX on display.

There are no restrictions to the security as required during the March 14th committee meeting during which it was mentioned that any project Manager can view any Project data – this feature will be useful for the functionality of request of transfer of funds.
